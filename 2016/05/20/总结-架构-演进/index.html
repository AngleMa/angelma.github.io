伸缩性

定义:

  • 指的是系统能够通过增加(减少)自身资源规模的方式增强(减少)自己计算处理事务的能力.
  • 如果这种增减是成正比的,就被称为线性伸缩性
  • 在网站架构中,通常指的是利用集群的方式增加服务器数量/提高系统的整体事务吞吐能力

    功能进行物理分离实现伸缩:

    定义:
    不同的服务器部署不同的服务,提供不同的功能

形式:

  1. 第一阶段服务器分层,新增服务器从现有服务器中分离出部分功能和服务

  2. 第二阶段服务纵向分离:将业务处理流程上的不同部分分离部署,实现系统伸缩性

  3. 第三阶段服务横向分离:将不同的业务模块分离部署,实现系统伸缩性

单一功能通过集群实现伸缩:

集群内的多台服务器部署相同的服务,提供相同的功能

应用服务器集群伸缩性

形式:

  1. HTTP重定向负载均衡
    • 原理:
      • 普通服务器进行负载均衡算法+302重定向
    • 优点:简单
    • 缺点:
      • 需要两次请求服务器才能完成一次访问,性能差
      • 单点故障:重定向服务器
      • 使用 http302状态码,有可能使得搜索引擎判断为SEO作弊,降低搜索排名
    • 应用:不常用

  1. DNS域名解析负载均衡
    • 原理
      • DNS服务器+负载均衡
    • 优点
      • 将负载均衡功能交给 DNS,省掉网站运维负载均衡服务器的费用
      • DNS支持CDN的基于地理位置的域名解析,会将域名解析为距离用户最近的一个服务器地址,改善性能
    • 缺点
      • 缓存失效问题:机器已经下线了,但是即使修改了DNS 里的记录,也无法立即生效
      • 负载均衡的控制权在域名服务商那里.
    • 应用
      • 大型网站使用 DNS 域名解析作为第一级负载均衡的手段,解析得到一个提供负载均衡服务的内部服务器,这个内部负载均衡服务器再进行负载均衡.

  1. 反向代理负载均衡
    • 原理
      • 反向代理+缓存资源+负载均衡,工作在 HTTP协议层面,也叫应用层负载均衡
    • 优点
      • 负载均衡和反向代理集成在一起,部署简单
    • 缺点
      • 单点瓶颈
    • 应用

  1. IP 负载均衡
    • 原理
      • 修改请求目标地址
    • 优点
      • 在内核进程内完成数据分发,较反向代理负载均衡性能更好
    • 缺点
      • 单点瓶颈:请求和分发都需要经过负载均衡服务器
    • 应用

  1. 数据库链路层负载均衡
    • 原理
      • 负载均衡服务器只分发请求,真实物理服务器来响应并返回数据
      • 在通信协议的数据链路层修改 MAC地址,又称为三角传输模式:通过配置真实物理服务器集群所有机器虚拟 IP和负载均衡服务器 IP地址一致,从而达到不修改数据包的源地址和目的地址就可以进行数据分发目的;由于实际处理请求的真实物理服务器IP 和数据请求的 IP 一致,不需要通过负载均衡服务器进行地址转换,可将响应数据包直接返回给浏览器
    • 优点
      • 消除单点瓶颈
    • 缺点
    • 应用
      • 目前大型网站使用最广的手段
      • Linux 最好的开源产品是 LVS

算法

  1. 轮询:
    所有请求一次分发
  2. 加权轮询:
    根据应用服务器硬件性能,进行权重分配的轮询
  3. 随机:
    请求被随机分配到各个应用服务器
  4. 最少连接:
    记录每个应用服务器正在处理的连接数,将请求分发到最少连接的服务器上
  5. 加权最少连接
  6. 源地址散列:
    根据 IP 进行 hash 运算,根据服务器个数取余,得到应用服务器,这样同一个IP地址的请求总是落在同一个服务器上进行处理,在一个会话周期内重复使用请求的上下文,实现会话黏滞
分布式缓存集群伸缩性
访问模型:

余数hash:
node_id = hash_key % len(nodes)
服务器列表下标 =缓存数据 key 的 hashcode % 服务器数目
一致性 hash 算法:
  1. 构造一个长度为0-2^23的整数环–一致性hash 环
  2. 将机器通过hash算法映射到环上(机器的hash计算是采用机器的IP或者机器唯一的别名作为输入值)—–根据节点名称的hash 值,将缓存服务器节点放置在这个 hash 环上
  3. 然后根据数据的Key值计算得到其Hash值（其分布也为[0, 232-1]），
  4. 接着在Hash环上顺时针查找距离这个Key值的Hash值最近的服务器节点，完成Key到服务器的映射查找。

扩展性

定义

在对现有系统影响最小的情况下,系统功能可持续扩展及提升的能力.

表现

  • 系统基础设施稳定不需要经常变更,应用之间较少依赖和耦合,对需求变更可以敏捷响应
  • 是系统架构设计层面的开闭原则

形式

1. 分布式消息队列来解耦

2. 分布式服务来复用

3. 可扩展的数据结构方便扩展字段

4. 开放平台来创造价值
